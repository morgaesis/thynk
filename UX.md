# Thynk UX Direction

## Experience Goal

Thynk should feel calm, fast, and work-focused. The first screen is the workspace itself: navigation on the left, writing in the center, and contextual tools only when they help. The interface should support quick capture without hiding the paths to deeper organization.

## Core Interaction Model

- Sidebar-first navigation with notes, folders, favorites, recent notes, tags, trash, graph, calendar, templates, notifications, and settings.
- Distraction-free editor with a clear title field, Markdown-aware rich editing, code blocks, tables, task lists, wiki-links, mentions, and slash commands.
- Command palette for keyboard-first search, switching, and creation.
- Autosave that feels trustworthy: clear saved/unsaved/saving states, local buffering, and no typing lag.
- New note creation that is immediate and focuses the title for fast rename.
- Graph and calendar as workspace views, not separate products.
- Settings as a modal surface for theme, shortcuts, editor preferences, AI keys, sync, and collaboration options.

## UX Principles

- No jank: typing, scrolling, opening notes, and switching views should never feel delayed.
- No layout surprise: focus states, code block controls, and editor affordances should not cause vertical jumps.
- Keyboard and pointer parity: power users can stay on the keyboard, while visible controls remain discoverable.
- Progressive disclosure: advanced features such as AI model selection, sync, permissions, and audit tools appear only when configured or relevant.
- Confidence over ceremony: autosave, locks, presence, and conflict states should be obvious without interrupting ordinary writing.
- Portable by default: users should understand that their content remains theirs.

## Visual Direction

- Quiet, dense, professional workspace layout.
- Strong typography in the editor, restrained chrome around it.
- Light and dark themes with tasteful defaults.
- Icon-led controls for common actions.
- Compact panels and modals for secondary workflows.
- Avoid marketing-page composition inside the product; the app should open directly into useful work.

## Important Workflows

- Capture a note quickly, rename it, and keep typing.
- Find any note or action with the command palette.
- Link notes with `[[wiki-links]]` and navigate backlinks.
- Move from notes to structured planning with properties, tasks, tables, and board or calendar views.
- Review knowledge through graph, tags, favorites, recent notes, and daily notes.
- Collaborate with presence, mentions, notifications, page locks, and eventually shared workspaces.
- Use AI only when the user has configured a provider or local model.
